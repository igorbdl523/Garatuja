#GARATUJA ( ͡ಠ ͜ʖ ͡ಠ)

Anotações – Programação (1º trimestre) Webdesign (revisão)

webdesign = criação visual de sites

importante:

layout (organização) cores + fontes responsivo (cel + pc) usabilidade (fácil de usar) HTML (eu do futuro)

html = estrutura do site

usa:

h1 → título p → texto img → imagem a → link

obs: não é linguagem de programação

CSS (eu do futuro)

css = estilo do site

faz:

cores tamanhos fontes layout (flex, grid)

resumo: html = base css = beleza

JS (eu do futuro)

js = comportamento do site

faz:

clique botão animação validação

resumo: js = interatividade

simples organizado básico funcionando

versão pessoal:

mais bonito estilo próprio mais info criatividade

diferença: prof = básico certo pessoal = completo + estilizado

Java vs JS vs TS

Java:

linguagem forte backend / apps compilada

JS:

web roda no navegador

TS:

JS melhorado tem tipos (mais seguro)

resumo: java ≠ javascript ts = js com tipo

POO (programação orientada a objetos) oq é

programar usando objetos

vantagens:

organização reutilização fácil manutenção classe

molde (modelo)

objeto

instância da classe

ex: classe carro → meuCarro

atributo

dados do objeto

ex: cor, nome, idade

método

ação do objeto

ex: andar(), falar()

this

this = objeto atual

não usa nome tipo a, b, c sempre usa this

construtor

roda quando cria objeto

serve pra:

iniciar valores

resumo: obj já nasce pronto

herança

uma classe herda da outra

ex: animal → cachorro

vantagem: reaproveita código

encapsulamento

proteger dados

usa:

public → tudo acessa private → só classe protected → classe + filhas

resumo: não deixar bagunçar os dados

getters e setters

getter: pega valor

setter: altera valor com regra

vantagem: controle total

get / set (forma moderna)

parece atributo normal

ex: obj.nome

mas por trás tem validação

polimorfismo

mesmo método, comportamentos diferentes

ex: calcularArea() muda dependendo da forma

resumo: mesmo nome → resultado diferente

static

pertence à classe

não precisa criar objeto

uso: Classe.metodo()

resumo: não usa this

revisão final (decorar)

classe → molde objeto → instância atributo → dado método → ação this → objeto atual construtor → inicia herança → herda encapsulamento → protege getter/setter → controla polimorfismo → muda comportamento static → classe
